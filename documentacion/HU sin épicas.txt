
REGLA TRANSVERSAL DE TESTING (ESENCIAL)
En todo el proyecto se ejecutarán únicamente tests críticos para el correcto funcionamiento del sistema.
Se evita a toda costa realizar testing opcional o de baja prioridad.



HU-01 — Registro de candidato
Como candidato
Quiero registrarme en la plataforma
Para poder postular a ofertas laborales
Criterios de aceptación:
Se puede ingresar: nombre, apellido paterno, apellido materno, correo, celular, rut, nro. documento, región y comuna como obligatorios; segundo nombre y calle como opcionales
Se valida formato de email
Se valida lógica numérica de RUT de Chile
No permite emails duplicados
El registro crea una cuenta tipo "candidato"
Se muestra error si falla el registro
Si falta un campo obligatorio no permite avanzar y muestra "Completa este campo"





HU-02 — Registro de empresa
Como empresa
Quiero registrarme en la plataforma
Para poder publicar ofertas laborales
Criterios de aceptación:
Se puede ingresar email, contraseña y nombre de empresa
No permite emails duplicados
Se crea cuenta tipo "empresa"
Se valida campos obligatorios
Manejo de errores visible





HU-03 — Login
Como usuario (candidato o empresa)
Quiero iniciar sesión
Para acceder a mis funcionalidades
Criterios de aceptación:
Login con email y contraseña
Retorna token JWT válido
Manejo de credenciales incorrectas
Persistencia de sesión en cliente
Logout elimina sesión
En app móvil de postulantes, tras 4 intentos fallidos se bloquea el acceso y se deriva al flujo de recuperación de contraseña
En app móvil de postulantes (MVP), el flujo de recuperación por correo es mockup: no se envían correos reales





HU-04 — Crear perfil de candidato
Como candidato
Quiero completar mi perfil
Para registrar mis datos y habilidades declaradas
Criterios de aceptación:
Puede ingresar:
skills
presentación personal
pretensión de renta
Validación de campos
Persistencia correcta
Las skills del perfil no se usan para calcular el porcentaje de matching; ese porcentaje usa CV limpio frente al contenido de la oferta, considerando además modalidades preferidas y cargas horarias preferidas del postulante





HU-05 — Editar perfil
Como candidato
Quiero editar mi perfil
Para mantener mis datos actualizados
Criterios de aceptación:
Puede modificar todos los campos
Los cambios se guardan correctamente
No se pierden datos previos





HU-06 — Subir CV
Como candidato
Quiero subir mi CV en PDF
Para que el sistema analice mi perfil automáticamente
Criterios de aceptación:
Solo acepta PDF
Tamaño máximo 1 MB
Se almacena correctamente
Se procesa posteriormente por IA
Manejo de error si archivo inválido





HU-07 — Calcular compatibilidad (HU importante/core, priorizar)
Como candidato
Quiero ver un porcentaje de compatibilidad con una oferta
Para saber qué tan adecuado soy para el puesto
Criterios de aceptación:
Se muestra porcentaje (0–100%)
Se calcula usando IA a partir del CV limpio frente al contenido de la oferta, considerando además modalidades preferidas y cargas horarias preferidas del postulante
No usa skills declaradas en el perfil
No usa presentación personal del perfil
No usa rango de sueldo esperado
No usa sueldo ofrecido de la oferta
No usa años de experiencia requeridos por la oferta
No usa años de experiencia declarados por el postulante dentro del CV
No usa nacionalidad del postulante
Manejo de error si falla IA





HU-08 — Explicación del match (HU importante/core, priorizar)
Como candidato
Quiero entender por qué tengo cierto nivel de compatibilidad
Para tomar mejores decisiones al postular
Criterios de aceptación:
Resume coincidencias o temas relevantes entre el CV y la oferta, sin tratar la lista de skills del perfil como fuente del score
Muestra razones del score alineadas con ese criterio
Información clara y legible
No muestra datos sensibles

HU-23 — Marcar postulación como vista
Como empresa
Quiero marcar una postulación como vista

Existe opción para guardar una oferta (botón o acción)

El usuario puede acceder a una sección “Guardadas”
El usuario puede quitar una oferta de guardadas
No se duplica una oferta si se guarda más de una vez
